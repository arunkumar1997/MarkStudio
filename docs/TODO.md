* **T-110 · External file-change reconciliation** — External edits to an open `.md` file (revert, `git pull`, another editor's save, an external formatter) now update the editor without jumping the cursor to the top. Because MarkStudio is a `CustomTextEditorProvider`, VS Code already keeps the `TextDocument` in sync with disk and fires `onDidChangeTextDocument`; the provider's existing echo-guarded handler already forwards that as `setContent`. The fix is in `src/webview/editor/createEditor.ts`: `setContentFromHost` reconciles the incoming text as a **minimal single-span diff** (new module-private `computeMinimalChange` trims the common prefix + suffix) instead of a full-document replace, so CM6 maps the existing selection across the change and the cursor is preserved when the edit is elsewhere. No separate `FileSystemWatcher` is added — for a text-backed custom editor `onDidChangeTextDocument` is VS Code's reconciliation surface; a raw watcher would double-fire on clean docs and risk clobbering unsaved edits on dirty ones. New **ADR-0009** records the decision and refines ADR-0004 for the text-backed case. No new dependency, no protocol change. *(2026-06-27 — see [CHANGELOG.md](CHANGELOG.md))*
